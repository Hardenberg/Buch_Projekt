# Kapitel 3 – Hashwerte, die sich erinnern

Jonas erzeugte die Hashwerte erneut.
Gleiche Datei. Gleiches System. Gleiche Parameter.

Das Ergebnis war anders.

Nicht stark. Nur vier Bits verschoben.
Er wiederholte den Vorgang.

Diesmal stimmte der Hash mit dem ersten Ergebnis überein.

Er startete eine automatisierte Serie von Prüfsummen. Dreißig Durchläufe.
Zwei Werte wechselten sich in einem stabilen Rhythmus ab.

Kein zufälliges Bitflipping.
Kein Speicherfehler.
Keine thermische Drift.

Die Abweichung war reproduzierbar.

Jonas exportierte die Werte und legte sie übereinander.
Die Differenzen bildeten eine Sequenz.

Keine willkürliche Verteilung.

Eine Struktur.

Und sie wiederholte sich.
